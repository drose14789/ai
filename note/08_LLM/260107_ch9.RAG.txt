Retrieval Augmented Generation(RAG)
[ RAG의 의미 ]
LLM 어플리케이션 개발에서 중요한 개념
­ LLM 등장 전 NLP에서부터 사용됨
­ 검색 증강 생성

1. Retrieval
(1) 데이터를 가져오는 것
(2) 구체적으로는 “컴퓨터 시스템에 저장된 자료를 취득하는 것”이라는 뜻도 있음
(3) 언어 모델이 가지고 있지 않은 정보를 가져오는 것
① 언어모델이 아웃풋을 만드는데 필요한 정보를 제공하는 것
② 언어모델이 “답변생성”에 능숙하지만, 답변 생성을 위한 모든 정보를 가지고 있지는 않음 
(ex) 보안이 걸려있는 사내자료
(ex) GPT4는 23년 10월까지 정보를 가지고 있음/GPT5는 24년6월

2. Augmented
(1) AR/VR에 사용되는 것과 같은 단어
(2) 마치 사실인 것처럼.
(3) Retrieval된 데이터를 LLM에게 주면서, “마치 이 정보를 아는 것처럼” 

3. Generation
(1) 생성
(2) 내가 데이터를 제공할테니 이 정보를 아는 것처럼 “답변을 생성하라”

[ 사용자가 원하는 정보 ]
- vector store (vector Database) : Rag 구성의 핵심
1. 사용자의 질문과 관련있는 데이터
(1) 관련이 있다는 것을 어떻게 판단할까?
(2) 관련성 파악을 위해 vector(토크을 1차원 배열로 변환한 값)를 활용함.
    ① 단어 또는 문장의 유사도를 파악해서 관련성을 측정함
    ② 유사도 계산 방법 : https://www.pinecone.io/learn/vector-similarity 

2. Vector를 생성하는 방법
(1) Embedding 모델을 활용해서 vector(1차원 행렬)를 생성함
(2) 문장에서 비슷한 단어가 자주 붙어 있는 것을 학습
  ex1. 왕은 왕자의 아버지다
  ex2. 여왕은 왕자의 어머니다
(3) 위의 두 문장의 유사도를 계산해 본다.
(4) Embedding Projector : https://projector.tensorflow.org/
